# PRD — 공공 직군 행정업무 슈퍼앱

> 기능 요구서 (Product Requirements Document)
> 최종 수정: 2026-07-01

## 1. 개요

공공 직군(행정) 실무자의 반복 업무를 한 곳에서 처리하는 웹 기반 슈퍼앱이다.
기존에 개별 도구로 흩어져 있던 팀 일정 공유, 엑셀 가공, 민원 응대, 뉴스 모니터링을
단일 앱으로 통합해 업무 전환 비용을 줄이는 것이 목표다.

- **대상 사용자**: 공공기관 행정 담당자, 팀장, 민원 대응 담당자
- **핵심 가치**: 반복 업무 자동화(RPA), 정보 공유, AI 보조
- **범위(초기)**: 아래 4개 기능 + 기존 전자결재/공지 모듈

## 2. 공통 요구사항

| 구분 | 요구사항 |
|------|----------|
| 인증 | 초기 버전은 이름 기반 식별. 추후 기관 SSO/계정 체계로 확장 |
| 브라우저 | 최신 Chrome/Edge 기준 |
| 언어 | 한국어 UI, UTF-8 |
| 데이터 | 모든 업무 데이터는 SQLite에 저장(파일 산출물 제외) |
| 접근성 | 사내망 환경 우선, 외부 API는 화이트리스트로 관리 |

## 3. 기능별 요구사항

### 3.1 팀원 스케줄 관리 (팀 캘린더)

팀원들의 **휴가·근무·출장** 등 일정을 등록/공유하는 캘린더.

**사용자 스토리**
- 팀원으로서, 내 휴가/출장 일정을 등록해 팀에 공유하고 싶다.
- 팀장으로서, 특정 날짜에 누가 자리를 비우는지 한눈에 보고 싶다.

**기능 요구사항 (FR)**

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| SCH-1 | 일정 등록: 제목, 유형(휴가/근무/출장/기타), 시작일, 종료일, 담당자, 메모 | 必 |
| SCH-2 | 월간/주간 캘린더 뷰로 팀 전체 일정 조회 | 必 |
| SCH-3 | 유형별 색상 구분 표시 | 必 |
| SCH-4 | 담당자·유형·기간으로 필터링 | 選 |
| SCH-5 | 일정 수정/삭제 | 必 |
| SCH-6 | 특정 날짜 클릭 시 해당일 일정 목록 표시 | 選 |

**데이터 항목**: `id, title, type, start_date, end_date, owner, memo, created_at`

**검증 규칙**: 종료일 ≥ 시작일. 유형은 정해진 enum만 허용.

### 3.2 엑셀 업무자동화 (분할/병합)

특정 **컬럼 기준**으로 엑셀 파일을 나누거나 여러 파일을 합치는 기능.

**사용자 스토리**
- 담당자로서, 부서 컬럼을 기준으로 하나의 엑셀을 부서별 파일로 자동 분리하고 싶다.
- 담당자로서, 여러 부서가 보내온 엑셀을 하나로 합쳐 취합하고 싶다.

**기능 요구사항 (FR)**

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| XLS-1 | 파일 업로드(.xlsx/.xls) 및 시트/컬럼 미리보기 | 必 |
| XLS-2 | **분할**: 선택한 기준 컬럼의 값별로 시트/파일 분리 | 必 |
| XLS-3 | 분할 결과를 여러 시트의 단일 파일 또는 ZIP(파일 여러 개)로 다운로드 | 必 |
| XLS-4 | **병합**: 여러 업로드 파일을 행 방향으로 결합(헤더 정합성 검사) | 必 |
| XLS-5 | 헤더 불일치·빈 컬럼 등 오류를 사전 검증하고 안내 | 必 |
| XLS-6 | 처리 이력(파일명, 작업유형, 행수, 일시) 기록 | 選 |

**제약**
- 업로드 최대 크기: 20MB (설정 가능)
- 개인정보 포함 가능 → 처리 후 서버 임시파일 즉시 삭제, 결과만 스트리밍 다운로드

**데이터 항목(이력)**: `id, job_type(split/merge), source_files, key_column, row_count, created_at`

### 3.3 민원대응 챗봇

첨부된 **민원 매뉴얼**을 근거로, 대응 방식이나 응대 스크립트를 작성해 주는 AI 챗봇.

**사용자 스토리**
- 민원 담당자로서, 매뉴얼을 근거로 한 표준 응대 스크립트를 즉시 받고 싶다.
- 담당자로서, 답변이 매뉴얼의 어느 부분에 근거했는지 확인하고 싶다.

**기능 요구사항 (FR)**

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| BOT-1 | 매뉴얼 문서 업로드/등록(PDF·TXT·MD) | 必 |
| BOT-2 | 민원 상황을 입력하면 매뉴얼 근거 대응 방식·응대 스크립트 생성 | 必 |
| BOT-3 | 답변에 **근거(인용) 구간** 표시 | 必 |
| BOT-4 | 멀티턴 대화(후속 질문에서 맥락 유지) | 必 |
| BOT-5 | 매뉴얼에 없는 내용은 "매뉴얼에 근거 없음"으로 명확히 고지 | 必 |
| BOT-6 | 대화/생성 스크립트 복사·저장 | 選 |

**AI 사양**
- 모델: **Claude Opus 4.8** (`claude-opus-4-8`)
- 방식: 매뉴얼 전문을 컨텍스트로 제공(문서 Q&A). 매뉴얼이 큰 경우 **프롬프트 캐싱**으로 비용 절감
- 인용: Claude **Citations** 기능으로 근거 구간 반환
- 스트리밍 응답으로 체감 응답성 확보

**비기능 요구사항**
- 매뉴얼·민원 내용은 민감정보일 수 있음 → 로깅 시 마스킹, API 키는 환경변수로만 관리
- 환각 억제: 시스템 프롬프트에서 "매뉴얼 근거 외 답변 금지" 명시

### 3.4 뉴스 기사 수집

공공 행정 관련 뉴스를 **매일 아침 자동 수집**하는 기능.

**사용자 스토리**
- 담당자로서, 출근 시 우리 업무 관련 최신 뉴스를 모아 보고 싶다.
- 팀장으로서, 키워드별로 걸러진 기사만 확인하고 싶다.

**기능 요구사항 (FR)**

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| NEWS-1 | 매일 지정 시각(기본 07:00)에 자동 수집 스케줄 | 必 |
| NEWS-2 | 등록된 뉴스 소스(RSS 등)에서 기사 수집 | 必 |
| NEWS-3 | 키워드 필터링(예: "행정", "공공", 기관명) | 必 |
| NEWS-4 | 수집 결과 목록 조회(제목, 출처, 링크, 수집일) | 必 |
| NEWS-5 | 중복 기사 제거(URL/제목 기준) | 必 |
| NEWS-6 | 수동 "지금 수집" 실행 | 選 |
| NEWS-7 | (선택) AI 요약: 수집 기사 일괄 요약 브리핑 | 選 |

**데이터 항목**: `id, title, url, source, published_at, collected_at, keyword`

**제약**
- 외부 사이트 수집은 로봇 배제/약관 준수, 요청 간격 조절
- 소스·키워드는 설정으로 관리(코드 수정 없이 변경 가능하도록 지향)

## 4. 범위 밖 (Out of Scope) — 초기 버전

- 계정/권한 관리(SSO), 조직도 연동
- 모바일 앱(반응형 웹으로 일부 대응)
- 뉴스 기사 원문 저장(링크만 저장)
- 엑셀 외 포맷(CSV는 후속 고려)

## 5. 성공 지표 (초안)

- 팀 일정 등록/조회 사용률
- 엑셀 분할·병합 월 처리 건수 및 수작업 대비 시간 절감
- 챗봇 응대 스크립트 채택률, "근거 없음" 오답 감소
- 뉴스 자동 수집 성공률(스케줄 실행 성공/전체)
