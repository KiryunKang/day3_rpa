# Operation — 공공 직군 행정업무 슈퍼앱

> 실행 방법 · 자주 발생하는 에러 · 사용법 (운영 가이드)
> 최종 수정: 2026-07-01

## 1. 사전 준비

| 도구 | 확인 명령 | 기대 버전 |
|------|-----------|-----------|
| Node.js / npm | `node -v` / `npm -v` | v24+ / 11+ |
| uv | `uv --version` | 0.11+ |
| (챗봇용) API 키 | `echo $OPENAI_API_KEY` | 발급 필요 |

> Python은 uv가 관리한다. 별도 시스템 Python 설치 불필요(`uv run`이 3.14 자동 사용).

### 환경변수 설정

민원 챗봇을 쓰려면 OpenAI API 키가 필요하다. `backend/.env`에 설정하면 앱 시작 시 자동 로드된다(`python-dotenv`).

```bash
# backend/.env  (git에 커밋 금지 — .gitignore 처리됨)
OPENAI_API_KEY=sk-...
# OPENAI_MODEL=gpt-4o   (선택, 기본 gpt-4o-mini)

# 또는 PowerShell 현재 세션 환경변수
$env:OPENAI_API_KEY = "sk-..."
```

> 키가 없어도 스케줄/엑셀/캘린더 기능은 정상 동작한다. 챗봇·AI 요약만 비활성.

## 2. 실행 방법

### 백엔드 (터미널 1)

```bash
cd backend
uv sync            # 최초 1회: 의존성 설치
uv run python main.py         # 개발 서버 → http://127.0.0.1:8000
```

- API 문서(Swagger): http://127.0.0.1:8000/docs
- 헬스 체크: http://127.0.0.1:8000/api/health

### 프론트엔드 (터미널 2)

```bash
cd frontend
npm install        # 최초 1회
npm run dev        # → http://localhost:5173
```

브라우저에서 **http://localhost:5173** 접속. `/api` 요청은 Vite 프록시를 통해 8000 포트로 전달된다.

### 프로덕션 빌드

```bash
cd frontend && npm run build     # dist/ 정적 자산 생성
# 백엔드에서 dist/를 정적 서빙하거나, 별도 웹서버(nginx)로 배포
```

## 3. 기능별 사용법

### 3.1 팀 스케줄
1. 상단 탭 **스케줄** 선택
2. "일정 등록" → 유형(휴가/근무/출장/기타)·기간·담당자 입력 → 저장
3. 캘린더에서 유형별 색상으로 확인, 담당자/유형 필터 사용

### 3.2 엑셀 자동화
- **분할**: 파일 업로드 → 미리보기에서 기준 컬럼 선택 → "분할 실행" → 결과 다운로드(다중 시트 또는 ZIP)
- **병합**: 여러 파일 업로드 → 헤더 검사 통과 시 "병합 실행" → 합쳐진 xlsx 다운로드
- 헤더 불일치 시 어떤 컬럼이 다른지 안내 메시지 표시

### 3.3 민원 챗봇
1. **매뉴얼 등록**: PDF/TXT/MD 업로드
2. 채팅창에 민원 상황 입력(예: "주민등록등본 대리 발급 문의 응대")
3. 매뉴얼 근거 대응 방식·응대 스크립트가 **근거 인용**과 함께 생성
4. 후속 질문으로 이어서 대화 가능. 매뉴얼에 없으면 "근거 없음" 고지

### 3.4 뉴스 수집
- 매일 07:00 자동 수집(스케줄러). 즉시 확인은 "지금 수집" 버튼
- 키워드/기간으로 필터, 제목 클릭 시 원문 링크 이동

## 4. 자주 발생하는 에러 & 해결

### 4.1 환경/실행

| 증상 | 원인 | 해결 |
|------|------|------|
| `python --version`이 "Python"만 출력 | Windows Store 스텁(가짜 python) | 무시. 항상 `uv run` 사용 |
| `No pyproject.toml found` | backend 밖에서 uv 실행 | `cd backend` 후 실행 |
| 포트 8000 `address already in use` | 이전 서버 미종료 | 아래 "포트 정리" 참조 |
| FE에서 `/api` 404/CORS | 백엔드 미기동 또는 프록시 미설정 | 백엔드 실행 확인, `vite.config.ts` proxy 확인 |
| 한글이 콘솔에서 깨짐 | PowerShell 콘솔 인코딩 | API 응답 자체는 UTF-8 정상. 브라우저/`/docs`에서 확인 |

**포트 정리 (PowerShell)**
```powershell
Get-NetTCPConnection -LocalPort 8000 -State Listen |
  ForEach-Object { Stop-Process -Id $_.OwningProcess -Force }
```

### 4.2 민원 챗봇 (OpenAI)

| 증상 | 원인 | 해결 |
|------|------|------|
| 503 "OPENAI_API_KEY 미설정" | 키 없음 | `backend/.env`에 `OPENAI_API_KEY` 설정 후 재시작 |
| 401 인증 실패 | 키 오타/폐기 | 키 재확인 또는 재발급 |
| 502 "OpenAI API 오류" | 모델 권한 없음/요청 초과 등 | `OPENAI_MODEL` 확인(기본 `gpt-4o-mini`), 잠시 후 재시도 |
| 답변이 매뉴얼과 무관 | 근거 강제 부족 | 시스템 프롬프트에 "매뉴얼 근거 외 답변 금지" 유지 |

> 참고: 챗봇은 OpenAI `chat.completions`를 사용하며 매뉴얼 전문을 시스템 프롬프트에 근거로 넣는다.
> `.env`는 `python-dotenv`로 앱 시작 시 자동 로드된다.

### 4.3 엑셀

| 증상 | 원인 | 해결 |
|------|------|------|
| 업로드 실패(413/거부) | 파일 크기 초과 | 20MB 이하로 분할 업로드 |
| `openpyxl`/엔진 오류 | .xls(구버전) 미지원 | .xlsx로 변환 후 업로드 |
| 병합 시 헤더 불일치 오류 | 컬럼 구성 상이 | 파일들의 헤더를 동일하게 맞춤 |
| 분할 결과 비어있음 | 기준 컬럼에 값 없음/오타 | 미리보기에서 올바른 컬럼 선택 |

### 4.4 뉴스 수집

| 증상 | 원인 | 해결 |
|------|------|------|
| 수집 0건 | 소스 URL 오류/키워드 과협소 | 소스·키워드 설정 점검 |
| 스케줄 미실행 | 앱이 꺼져 있었음 | 서버 상시 구동 필요(운영은 서비스/데몬화) |
| 중복 기사 | URL 상이한 동일 기사 | URL UNIQUE 기준. 필요 시 제목 정규화 추가 |
| 외부 사이트 차단 | 과도한 요청/차단 | 요청 간격 조절, RSS 우선 사용 |

## 5. 운영 팁

- **DB 백업**: `backend/app.db` 파일을 주기적으로 복사(서버 중지 상태 권장)
- **DB 초기화**: `app.db` 삭제 후 재시작하면 스키마·샘플 재생성(운영 데이터 유실 주의)
- **로그**: 민원/개인정보는 원문 로깅 금지. 필요한 식별자만 기록
- **상시 구동**: 뉴스 스케줄러는 서버가 켜져 있어야 동작 → 운영 시 `nssm`(Windows 서비스)/`systemd` 등으로 데몬화
- **키 회전**: API 키 노출 시 즉시 폐기·재발급. `.env`는 절대 커밋 금지
