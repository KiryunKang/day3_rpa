# git commit

git에 업로드할 때 사용하는 스킬입니다. 아래 4단계를 순차적으로 진행합니다.

## 1단계: 초기화
`gh auth status`로 GitHub에 로그인되어 있는지 확인합니다.
앞서 이미 확인했다면 이 단계는 건너뛸 수 있습니다.
로그인이 되어 있지 않다면 로그인을 시도하고, 필요 시 사용자에게 로그인을 요청합니다.

## 2단계: 레포지토리 확인
연결된 원격 레포지토리가 있는지 확인합니다.
없다면 새 레포지토리를 만들거나 사용자에게 기존 레포지토리 이름을 요청합니다.

## 3단계: 문제 확인
업로드 전 충돌이나 파일 오류가 없는지 확인합니다. (`git status`, 스테이징 대상 점검)

## 4단계: 업로드
`git commit` 후 `git push` 합니다.
사용자가 커밋 메시지를 지정하지 않으면 변경 내용을 요약해 알아서 작성합니다.

# 유의사항
  커밋 제목과 커밋 메시지는 반드시 한글로 작성합니다.
