# code review

FE 와 BE 개발이 끝난 후 코드를 리뷰하는 스킬입니다.
**반드시 사용자가 명시적으로 요청한 경우에만** 실행합니다.

대상 스택
- BE: Python + FastAPI
- FE: React + TypeScript + Vite

## 1단계: 리뷰 대상 확인
리뷰할 범위를 확인합니다. (변경된 파일, 특정 폴더, 전체 등)
지정이 없으면 사용자에게 범위를 물어봅니다.

## 2단계: 세 가지 관점으로 리뷰
각 파일/코드를 아래 3가지 기준으로 검토합니다.

1. **직관성** — 네이밍, 구조, 가독성, 함수/컴포넌트 책임 분리가 명확한가
2. **보안 취약성** — 입력 검증, 인증/인가, 시크릿 노출, SQL/XSS 등 주입, CORS 설정 등
3. **재사용성** — 중복 코드, 공통화 가능 여부, 결합도/응집도

## 3단계: 결과를 표로 보고
검사 결과를 아래 표 형식으로 직관적으로 표시합니다.

| 파일 | 관점 | 문제/개선점 | 심각도 | 제안 |
|---|---|---|---|---|

- 심각도는 높음/중간/낮음 으로 표기합니다.
- 문제가 없으면 "문제 없음"으로 명확히 표시합니다.

# 유의사항
- 이 스킬은 사용자가 명시적으로 요청했을 때만 사용합니다.
- 모든 리뷰 내용과 보고는 한글로 작성합니다.
- 임의로 코드를 수정하지 않고, 문제와 제안을 보고한 뒤 지시를 기다립니다.
